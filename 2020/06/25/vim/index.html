:ju(mps) 显示当前窗口的跳转列表，每个单独的窗口都有一份自己的跳转列表。

数字操作

当对 007 执行 Ctrl-a 操作时，默认情况下会变成 010，因为在默认情况下 vim 会认为以 0 开头的数字是八进制的数字，因此 007+1 = 010。

同理，vim 认为以 0x开头的数字是 16 进制的数字，因此 0x09+1=0x0a。

使用 set nrformats=xx可以改变 vim 的缺省行为，使用 help nrformats可以查阅支持的选项。

终端位置

打开路径

终端打开之后会默认进入当前vim窗口的工作路径，即在vim中输入 :pwd 的结果。

  • 通过 :lcd {path} 命令可以修改当前vim窗口的工作路径，如果 path 为空则进入 home 目录；
  • 通过 :cd {path} 命令有同样的作用，区别在于 lcd 命令只改变当前窗口，cd 会改变当前屏幕所有窗口；
  • 通过 :cd %:h 命令进入当前文件所在的路径；

Ex Mode

关于 Ex 命令的历史

Unix下最初的文本编辑器是ed，然后经过几代的改进变成了ex，ex编辑器支持使用 :visual命令切换到屏幕编辑模式，简写为:vi，即vi编辑器名字的由来，vim表示vi improved，即vi编辑器的增强版。

进入Ex模式

在normal模式下，按下 :就进入 Ex 命令模式，按下<esc>切换回normal模式。

重复上一个 Ex 命令的方法是 @:。

escape 函数

用法：escape(@u,'?\/')；

作用：将 u 寄存器中的字符串中的 ?\/ 进行转义，返回转义之后的字符串。

  2. 输入 /V<c-r>=，然后输入 escape(@u,'/\')(正向搜索)，或 escape(@u,'?\')(反向搜索) ;

global命令

global可以在指定范围内的行，并匹配指定模式的行执行Ex命令。

根据vim的文档，global命令的分隔符可以是任意的 single-byte character ，比如 ! % + = , # _ 等。

命令格式

如果没有指定范围，global命令的作用范围是整个文件，即range默认为 %。

如果没有指定匹配模式，则默认使用查找历史中上一次的匹配模式。一般先在查找模式下进行正则匹配，确认匹配正确之后，在之后的global命令将pattern处留空（//不能省略），这样可以保证作用域的正确。

如果没有指定命令，则默认执行p(print)命令，cmd可以是global之外的任何Ex命令。

指定特定寄存器的方式为 "reg_name

分类

无名寄存器 "

当没有指定寄存器时，vim 将默认使用 " 寄存器；因此指定无名寄存器与否，效果相同。

x s c d y 等命令都会改变无名寄存器的内容。

复制寄存器 0

只有 y 命令才会改变复制寄存器的内容

黑洞寄存器 _

使用删除命令时，如果指定该寄存器，删除的内容将消失，不会保存在任何地方，当然也不会影响其他任何寄存器。

有名寄存器 a-z

通用寄存器，指定特定的寄存器用于保存特定的内容。

正则表达式

  • 元字符是具有特殊意义的字符，可以表达自身字符本身以外的含义。
  • 由于 vim 本质上还是一个编辑器，所以 vim 中的元字符与其他的编程语言中的元字符有一些不同，普通模式下的大部分元字符必须加上反斜杠进行转义才能起到元字符的作用。

特殊字符

  • 如果要搜索 /，在所有模式下都需要加上 \ 转移符；
  • 如果要搜索 \，经试验可以加上 \ 转移符，也可以不加，都可以搜索到\；

四种模式

magic 模式

  • 这是 vim 中的默认模式，也是 vim 推荐的用法。

  • 在 \m(magic) 模式下，除了三个元字符之外，其他的元字符都要加上反斜杠进行转义。

  • 自身可以直接表示正则表达式含义的元字符：

  • . 匹配的是任意单个的字符，而 * 匹配的是任意的数量。

  • . 也可以与其他表示数量的元字符使用，同样不用加反斜杠。

  • * 也可以与其他表示字符的元字符使用，也不用加反斜杠

    • 特殊之处在于，单独搜索 * 时，* 表示符号自身的含义；
    • 但是当 * 前面有字符时，* 将表示正则表达式中的数量；

nomagic 模式

  • \M 模式与 \m 模式很相近，唯一的不同是对待 . 的区别, \M 模式下将其视为普通的字符。

  • 自身可以直接表示正则表达式含义的元字符：

very magic 模式

  • 在此模式下，所有的元字符都具有特殊含义，在正则表达式中可以直接使用；
    • 一切 并不完全准确，只有字符类的元字符才可以不加反斜杠，字母类的元字符 (\w,\d,\s,\x) 还是要加上反斜杠。
  • 如果要表示字符本身的含义，则需要使用反斜杠；

very nomagic 模式

  • 在此模式下，将所有的字符视为其本身的含义，类似于fgrep的做法；
  • 如果要使用正则表达式，则一切元字符都 必须 加反斜杠，但是一般只会在不用正则表达式的情况下才会开启该模式。
